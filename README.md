# Java Design Patterns

This project contains practical implementations of classic Design Patterns using Java, focusing on clean code, good practices, and real-world applicability.

## 📚 Implemented Patterns

### 🔹 Singleton
Ensures that a class has only one instance and provides a global access point to it.

**Use cases:**
- Shared resources management
- Application configuration
- Database connections

---

### 🔹 Strategy
Defines a family of algorithms, encapsulates each one, and makes them interchangeable at runtime.

**Use cases:**
- Switching behavior dynamically
- Replacing complex `if/else` structures
- Following the Open/Closed Principle (SOLID)

---

### 🔹 Facade
Provides a simplified interface to a complex subsystem, making it easier to use.

**Use cases:**
- Reducing system complexity
- Encapsulating multiple components
- Improving readability and maintainability
